# character — 回复风格预设

本目录收录可供 AI 模型（或用户）直接引用的**回复风格上下文预设**。

## 设计目标

不同的 AI 模型具有各自鲜明的回复风格。当用户在某一平台（如 GitHub Copilot、Cursor、Windsurf 等）使用特定模型时，如果希望体验接近另一款模型的风格，可将对应预设文件的内容粘贴到对话的 **System Prompt** 或对话开头，以引导模型向目标风格靠拢。

## 使用方式

1. 打开目标风格对应的 `.md` 文件（见下表）。
2. 将文件全文复制，粘贴到以下任一位置：
   - **System Prompt**（推荐，持久生效）
   - 当前对话的**第一条消息**（临时生效）
3. 继续正常对话，模型将尽量按预设风格回复。

## 可用预设

| 文件 | 模拟风格 | 适合场景 |
|---|---|---|
| [`claude_style.md`](./claude_style.md) | Anthropic Claude | 注重分析深度、结构清晰、有温度的编程助手体验 |
| [`gpt_style.md`](./gpt_style.md) | OpenAI ChatGPT / GPT-4 | 简洁直接、代码优先、实用主义风格 |

## 贡献指南

如需添加新风格：
1. 参考现有预设文件的结构新建 `<风格名>_style.md`。
2. 在文件开头用一段话描述该风格的核心特征。
3. 提供若干**示范对话**（用户提问 + 理想回复格式示例），让模型能够上下文学习（in-context learning）。
4. 在本 `README.md` 的表格中添加对应条目。
